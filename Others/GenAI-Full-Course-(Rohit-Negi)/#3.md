### I Build ChatBot from Scratch | | GenAI Full Course #3 - (Rohit Negi)

#### Summary
This video provides a technical walkthrough on building domain-specific LLM-based chatbots, focusing on context management, token optimization, and system instruction engineering.

**1. Token Economics & Cost Optimization (1:24 - 11:20)**
* LLM cost scales linearly with total token count (input + output). Context window maintenance requires appending entire conversation histories.
* Strategies for context management to reduce costs include:
    * **Sliding Window:** Maintaining only the last N messages (e.g., last 50 messages).
    * **Hybrid History:** Storing a fixed set of initial messages (for foundational context) plus a sliding window of recent interactions.
    * **Summarization:** Utilizing the LLM to generate periodic summaries of long-context threads to compress state, although this risks information loss during the distillation process.

**2. System Role Configuration (12:00 - 28:00)**
* To ensure model persona adherence and constraint satisfaction, instructions must be injected via the **System Instruction** configuration layer rather than the message history content.
* This architectural approach is critical to prevent "jailbreaking" or context-forgetting, as the System Prompt remains globally persistent regardless of user input aimed at clearing context or overriding directives.

**3. Implementation Strategies (28:20 - 53:00)**
* **DSA Instructor Chatbot:** Built by defining rigid system instructions that limit the model to domain-specific queries. The implementation uses the `google-generative-ai` SDK and Node.js. It demonstrates a pattern where unauthorized queries (e.g., general knowledge) trigger a pre-defined aggressive refusal response.
* **Persona-based Chatbot (Ex-girlfriend context):** Demonstrates using a rich system prompt to simulate specific personality traits, conversational slang, and behavioral constraints (e.g., possessiveness, specific emotional responses). 
* **Technical Workflow:** The workflow utilizes an `API` call pattern where the `system_instruction` is decoupled from the `user` turn history. For frontend integration, the speaker recommends using standard `fetch` API calls with `curl` commands to interact with the LLM backend.
