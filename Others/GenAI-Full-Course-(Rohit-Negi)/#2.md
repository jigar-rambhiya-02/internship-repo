### How Does LLM Work | Code Implementation | | GenAI Full Course #2 - (Rohit Negi)

#### Summary
This video provides a conceptual breakdown of Large Language Models (LLMs) and a practical implementation using the Google Gemini API. 

**Core Theoretical Concepts:**
* **Next-Token Prediction:** LLMs do not "calculate" in the traditional sense; they operate on a fundamental probabilistic principle of next-token prediction based on learned patterns from massive training corpora (1:16 - 2:34).
* **Limitations of Pure Inference:** Without external tools, LLMs exhibit high failure rates in tasks requiring symbolic arithmetic or real-time data retrieval. For example, counting characters in "strawberry" or determining current weather often yields hallucinations due to the lack of internal execution engines (2:45 - 4:18).
* **The Tool-Use Paradigm:** Modern LLM agents circumvent these limitations by generating code (e.g., Python scripts) that are executed by external tool-use environments (interpreters/APIs), passing the computed result back to the model for final synthesis (8:49 - 10:32).

**Technical Implementation Details:**
* **API Integration:** The author demonstrates initializing a project with `npm` and utilizing the `@google/generative-ai` SDK. Communication is managed via API keys, which track usage quotas at the server level (18:01 - 21:09).
* **Context Window & History Management:** 
    * **Statelessness:** Pure LLM inferences are stateless. To maintain conversational history (context), the user must explicitly provide a message array (history) comprising previous `user` and `model` role turns (26:01 - 32:20).
    * **Automated Context:** The video contrasts manual array management with the `ai.chats.create()` method, which automates session history state, ensuring the model maintains context across turns without manual payload overhead (43:37 - 47:30).
* **Execution Flow:** Using `async/await` patterns with Node.js, the implementation wraps user input in a loop, facilitating an interactive chat interface that maintains persistent context through the session (38:48 - 40:01).
